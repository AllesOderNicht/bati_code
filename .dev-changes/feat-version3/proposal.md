# Proposal: BATI V2 结果页内容丰富化 + SSR/浓度视觉强化

## Goal
将 17 种梗人设的结果页从"简单展示"升级为"深度共情 + 有梗有料"的阅读体验。每个人设的文案都是独特的、有性格洞察深度的、带有通俗公司文化梗的内容。SSR 和浓度人设通过完全不同的视觉风格和专属内容制造"隐藏款解锁"的惊喜感。

## Background
当前 17 个人设的结果页内容量单薄——每个人设只有 1 段人格描述（约 80-120 字）和 1 段梗来源，缺乏让用户"被说中了"的共情深度，各人设间的差异化也不够明显。`reasonTemplate` 字段已有文案但从未展示。SSR 和浓度人设仅靠一个小徽章与普通人设区分。

## User Stories
- 作为测试完成的用户，我希望结果页的人格描述足够深入，让我觉得"这个测试怎么这么懂我"。
- 作为分享用户，我希望结果页有足够有趣的内容让朋友也想来测一测。
- 作为触发 SSR 的幸运用户，我希望整个结果页的视觉风格都和普通结果完全不同，让我有"抽到隐藏款"的兴奋感。
- 作为测出浓度结果的用户，我希望看到一个动态的浓度可视化，强化"99.99% 纯度"的仪式感。

## Technical Approach

### PersonaProfile 数据模型扩展
在现有字段基础上新增少量字段来承载差异化内容：
- `deepInsight`: 深层性格洞察（1 段，约 80-120 字，作为人格描述的第二段）
- `funComment`: 趣味点评（1 段，约 40-80 字，作为人格描述的第三段，轻松幽默的收尾）
- `dailyHabits`: 日常行为特征列表（3-4 条短句，用于丰富共情感知）
- `funFacts`: 趣味冷知识/公司文化梗（2-3 条，轻度玩梗，通俗易懂）
- `ssrExclusiveNote`: SSR/浓度人设专属内容（仅 SSR 和浓度人设有值）

### 17 种人设文案全量重写
- `personaDescription` 保持为核心性格描述（第一段），不缩减
- 新增 `deepInsight` 作为深层洞察段（性格本质、行为动机、深层逻辑）
- 新增 `funComment` 作为趣味收尾（轻松幽默，让人会心一笑）
- 新增 `dailyHabits` 作为"被说中了"的共情点
- 新增 `funFacts` 作为公司文化相关的有趣内容
- `memeOrigin` 扩充为更详细的梗解读
- `reasonTemplate` 融入结果页展示

### 结果页展示增强
保持现有 UI 结构框架，但在内容层面显著丰富：
- 人格描述区展示 3 段（核心 + 洞察 + 趣味点评）
- 日常行为特征以简洁列表形式展示
- 趣味冷知识以卡片形式展示
- `reasonTemplate` 融入"为什么像/梗解读"面板
- SSR/浓度人设显示专属内容区

### SSR 视觉重塑
SSR 人设（天选打工人、魔术师）的结果卡片采用完全不同的视觉风格：
- 暗金色主题：深色背景 + 金色/琥珀色文字和边框
- 浮动粒子或微光动效，制造"稀有"氛围
- 专属「隐藏档案」内容区块

### 浓度视觉强化
浓度人设（99.99% 阿里/字节/鹅）加入浓度可视化：
- 动态浓度进度条（渐变填充 + 动画）
- 整体配色根据公司气质微调（阿里橙、字节蓝、腾讯绿）
- 专属「纯度报告」内容区块

## Product Goals And Non-Goals

### In Scope (MUST)
- MUST: 扩展 PersonaProfile 类型，新增 deepInsight、funComment、dailyHabits、funFacts、ssrExclusiveNote 字段
- MUST: 全量重写 17 种人设的文案内容，每个人设文案独特且有深度
- MUST: 结果页展示扩充后的多段人格描述、日常特征和趣味冷知识
- MUST: 将 reasonTemplate 融入结果页展示
- MUST: SSR 人设整卡片改为暗金色主题风格
- MUST: 浓度人设加入浓度进度条和配色偏移
- MUST: 保持移动端友好的阅读体验

### Out of Scope (MUST NOT)
- MUST NOT: 改变现有的算分逻辑或人设匹配机制
- MUST NOT: 修改题库或答题流程
- MUST NOT: 添加新的页面路由或导航
- MUST NOT: 引入后端服务

### Optional (MAY)
- MAY: 为海报导出适配新的内容结构
- MAY: 为人设图鉴页面同步更新展示内容

| PH-2 | 结果页内容展示增强与 SSR/浓度视觉重塑 | 更新结果页组件展示扩充内容，实现 SSR 暗金主题和浓度进度条 |
