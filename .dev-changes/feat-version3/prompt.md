# Prompt

## 目标
丰富 BATI V2 的结果页内容，让每个人设类别（共 17 种）的结果页都有独特的、差异化的内容展示。

## 核心方向：文案质量升级，不加结构
不需要增加新的 UI 模块/区块。重点是把现有的内容字段（personaDescription、memeOrigin、keywords、reasonTemplate 等）写得更丰富、更长、更有层次，让每个人设的结果页读起来更有料。

## 共情方向：性格洞察型
增加的共情内容应该是深层的性格分析，让用户觉得"这个测试竟然说得这么准"。不是简单的日常场景罗列，而是对性格本质的精准描述和洞察。

## 梗的风格：轻度玩梗
用比较通俗的方式融入公司文化特点，不熟悉互联网的人也能看懂。不需要太多圈层黑话或内部梗，保持正向、有趣、易传播。

## reasonTemplate 处理
将 reasonTemplate 融入现有的"为什么像/梗解读"面板中，不新增区块但丰富现有内容。

## SSR 和浓度人设特殊处理
采用内容特权 + 视觉氛围双重强化：
- 内容层面：SSR/浓度人设比普通人设多专属内容（如隐藏档案、稀有度解读）
- 视觉层面：SSR 加入微妙发光/闪烁动效，浓度人设加入浓度条可视化

## 约束
- 保持现有设计风格（暖色调 clay 风格 + 手绘边框）
- 确保移动端阅读体验
- 17 个人设的文案必须各有特色，不允许模板化
- 正向玩梗边界，不攻击不讽刺
